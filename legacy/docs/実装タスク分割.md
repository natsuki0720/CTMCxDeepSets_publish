# 公開版再構成に向けたタスク分割（実装開始前）

## 1. 目的
`docs/公開向け再構築_要件整理.md` と `docs/公開版_実装仕様詳細設計.md` をもとに、実装着手前のタスクを依存関係つきで分割し、段階的に進めるための実行計画を定義する。

## 2. 前提（ドキュメント読込結果 + 追加合意）
- 初期公開は「公開向け最小構成（v0.1想定）」を対象とする。
- 研究用断片コードの持ち込みは避け、責務分離された `src/` + `scripts/` + `configs/` を基準に再構成する。
- 推論APIは `run_dir`（`model_config.yaml` + `weights/best_model.pt`）から再構築できる構成を必須とする。
- 指標は寿命（逆数変換後）MAEを主指標とし、副次情報は推論時間を扱う。
- **確定**: `real_data` は公開しない（配布時は仮想データまたはダミーデータ生成導線を提供）。
- **確定**: 学習済みモデルは CPU / GPU（CUDA）双方で推論可能なロード・実行手順を提供する。
- **確定**: READMEにクイックスタートを最終的に整備し、公開判定ゲートに含める。
- **確定**: 公開時に同梱する学習器は特定のDeepSetsモデルのみとし、Bayes系モデルは含めない。

## 3. タスク分割（フェーズ別）

### フェーズ0: 仕様凍結（実装前の合意形成）
**目的**: 手戻りを防ぐため、未確定事項を先に確定する。

- T0-1. 公開スコープ確定
  - **確定**: 公開時は特定のDeepSetsモデル1系統のみを同梱する。
  - `real_data` 非公開方針を仕様に反映（README・docsへ明記）。
  - 非公開データ代替として、仮想データ生成手順またはダミーデータ配布方針を決定。
- T0-2. 実行環境・再現性レベル確定
  - Pythonバージョン、CPU/GPU前提、再現性レベル（R1/R2/R3）を決定。
  - CPU/GPU間の互換推論要件（`map_location`、dtype、依存バージョン）を決定。
- T0-3. 受け入れ基準（公開判定ゲート）確定
  - 最低限の手動スモーク要件、READMEクイックスタート再現要件を定義。
  - CPUのみ環境でも実行可能な最小経路（短縮実行）を定義。

**完了条件（DoD）**
- 未確定事項の回答が `docs/` に追記され、実装中の判断保留項目がない。

---

### フェーズ1: リポジトリ骨格と設定基盤
**目的**: 実装を載せる土台を先に整える。

- T1-1. ディレクトリ骨格作成
  - `src/ctmc_surrogate/{data,features,models,train,eval,inference}`
  - `scripts/`, `configs/{dataset,model,train,experiment}`, `tests/`
- T1-2. 実行環境定義
  - `pyproject.toml` の最小依存定義。
  - CPU向け依存とCUDA利用時の注意点（インストール注記）を分離して記述。
- T1-3. 実験実行の共通規約
  - `run_dir` 連番運用（例: `EXP-0001`）実装方針の文書化。
  - 設定保存ルール（学習時 `model_config.yaml` 保存）を文書化。

**完了条件（DoD）**
- 最小構成で `python scripts/train.py --help` 相当のCLI雛形が起動する。

---

### フェーズ2: データ生成・特徴量変換
**目的**: 学習入力までの前処理を独立モジュールとして確立する。

- T2-1. CTMCデータ生成モジュール実装（`data/`）
  - `generate_dataset(config, seed)` を実装。
- T2-2. 特徴量変換モジュール実装（`features/`）
  - `build_features(samples, strategy)` を実装。
- T2-3. データ準備CLI実装（`scripts/prepare_data.py`）
  - 設定読込→生成→保存までのフローを実装。
  - 非公開 `real_data` 依存がなくても再現可能な入力生成導線を提供。

**完了条件（DoD）**
- 設定ファイル1つでデータ生成から特徴量保存まで完了する。

---

### フェーズ3: モデル・学習・評価
**目的**: 学習可能な最小ループと評価レポートを作る。

- T3-1. モデル定義実装（`models/`）
  - 注意機構付き `DeepSetsAttnRegressor` と `build_model(model_config)` を実装。
  - 既存利用箇所向けに `DeepSetsRegressor` エイリアスを維持。
- T3-2. 学習ループ実装（`train/` + `scripts/train.py`）
  - 学習開始時 `model_config.yaml` 保存。
  - 学習終了時 `weights/best_model.pt` / `metrics.json` 保存。
- T3-3. 評価実装（`eval/` + `scripts/evaluate.py`）
  - 主指標: 寿命MAE（逆数変換後）
  - 副次情報: 推論時間
  - 集計: データセット別 + 状態別（仮想データは全体平均も）

**完了条件（DoD）**
- 1つの実験設定で学習→評価まで実行し、`run_dir` に成果物が揃う。

---

### フェーズ4: 推論API・CLI（CPU/CUDA互換対応を含む）
**目的**: 学習成果物を利用可能な形で提供する。

- T4-1. 推論ローダー実装（`inference/`）
  - `load_predictor_from_run_dir(run_dir)`
  - `device` 指定（`cpu` / `cuda` / `auto`）を受け取り、ロード時 `map_location` を適用。
- T4-2. 推論API実装（`inference/`）
  - `predict(run_dir, input_data, output_mode="lifetime|raw")`
  - CUDA非対応環境では自動でCPUへフォールバックする方針を定義。
- T4-3. 推論CLI実装（`scripts/predict.py`）
  - `--run-dir` 指定を標準とし、`model_config.yaml` 最優先で再構築。
  - `--device {auto,cpu,cuda}` を提供。
- T4-4. 互換スモーク確認
  - CUDA環境で学習した重みをCPU環境で読み込む確認手順を整備。
  - CPU学習重みをCUDA環境で読み込む確認手順を整備。

**完了条件（DoD）**
- 同一入力に対して `output_mode=lifetime` と `output_mode=raw` の両方を取得できる。
- CPU保存重み / CUDA保存重みの双方が、少なくともCPU推論で再利用できる。

---

### フェーズ5: 公開ドキュメント整備
**目的**: 第三者再現可能な公開体裁を完成させる。

- T5-1. `README.md` 再構成
  - クイックスタート、学習/評価/推論手順、制約記載。
  - `real_data` 非公開であること、代替データ利用フローを明記。
  - CPU/CUDA両方の起動例を記載。
- T5-2. 補助ドキュメント整備
  - `CONTRIBUTING.md`, `CITATION.cff`, `LICENSE`（必要に応じて `CODE_OF_CONDUCT.md`）
- T5-3. `docs/` 相互参照整備
  - 要件・詳細設計・本タスク分割書のリンク統一。

**完了条件（DoD）**
- クリーン環境でREADME手順のみで最小実行フローを辿れる。
- CPU環境向けクイックスタートと、CUDA利用時の追記事項が区別して記載されている。
